PROJECTS

cloudblox/ Personal IaC platform. Production-grade AWS infra from a single blox.yaml. private · pre-launch
The platform layer between DevOps and developers. At scaling startups, 1-2 DevOps engineers field every infra request: devs can't self-serve, DevOps can't keep up. CloudBlox solves both: DevOps writes infra once, developers ship on every push.
// how it works
  • ▸Composable blocks (EKS, RDS, Lambda, ECS, static-sites, more) provision real AWS resources. Pulumi runs under the hood; devs never write or see Pulumi code.
  • ▸Two-repo model: infra repo (DevOps owns shared platform) and gitops repo (developers own workloads). One CLI, same cloudblox deploy flow for both.
  • ▸GitOps-friendly: every workload is declared in YAML, versioned, and reviewed in PRs. Rollback is a git revert.
